Descrizione del progetto:

DOMINATOR è un dispositivo - cronometro per sport come Airsoft, Paintball. Il cronometro utilizza un microcontrollore di controllo Arduino (Arduino Uno / Nano) o un chip separato (Atmel) AtMega328P, che controlla l'intera logica del sistema. Il cronometro include un display a caratteri LCD che misura 16x2 (16 caratteri x 2 linee) o 20x4, sul quale vengono visualizzati i tempi delle singole squadre. Ci sono 2 squadre nel gioco: squadra RED e squadra GRE che giocano l'uno contro l'altro e occupano il punto immaginario formato da questo dispositivo - DOMINATORE. All'inizio del gioco, il display mostra i tempi 00 ore, 00 minuti, 00 secondi per Squadra RED , Squadra GRE . Ogni membro della squadra è dotato di tessera, braccialetto o portachiavi, che comunica sulla frequenza 13,56MHz con un lettore RFID, se si trova nel raggio d'azione. Portachiavi e braccialetti possono essere codificati a colori (verde, rosso a seconda della squadra e della funzione ...), il che consente loro di essere distribuiti in modo più efficiente ai giocatori, ma anche di archiviarli e conservarli. Se la squadra ROSSA occupa un punto, il giocatore della squadra attacca la sua carta al lettore, occupando così il punto. Questa azione attiva un cicalino che annuncia un cambiamento nel punto. Quindi il LED della squadra ROSSA in questa stazione si accenderà e il display inizierà ad aggiungere tempo per quella squadra. Se una squadra GRE occupa un punto e un membro di quella squadra collega la carta al lettore, un cicalino suona di nuovo, il LED di quella squadra si accende, il tempo della squadra di RED viene messo in pausa e il tempo della squadra conta. GRE, che ha occupato il punto. Se la carta / il portachiavi è attaccato dall'organizzatore (arbitro), entrambi i tempi vengono sospesi fino a quando una delle squadre non inserisce la carta. Questo tipo di carta è quindi adatto per iniziare una pausa, ma può essere utilizzato anche a fine partita, quando è necessario fermare il tempo per valutare la partita. C'è ancora un quarto tipo di carta in DOMINATOR - gomma, che fa una pausa entrambe le volte e ripristina i valori iniziali come dopo aver avviato il cronometro DOMINATOR. Utilizzato per riavviare il gioco, inizializzare un nuovo gioco. Il progetto è adatto per eventi di Airsoft, impianti sportivi o squadre di Airsoft, che possono costruire l'attrezzatura. Il vantaggio del sistema DOMINATOR è che il sistema può essere clonato e creare X punti con la stessa configurazione tramite codice macchina, che possono essere combattuti indipendentemente nel gioco, mentre le carte RFID funzioneranno completamente su tutti i dispositivi DOMINATOR. Il sistema è testato, funzionale, di facile manutenzione. Il lettore NXP RC522 funziona a una frequenza di 13,56 MHz, registra tag RFID a circa 3 centimetri (è richiesto un contatto quasi fisico con il lettore), formato tag compatibile ISO / IEC 14443-A. Lo standard di tag fornito include anche ISIC, schede bus che funzionano come schede NFC e possono comunicare il loro indirizzo fisico - UID a un lettore nel raggio d'azione.

Hardware per il progetto - RFID DOMINATOR:

  • Arduino Uno/Nano
  • Lettore RFID NXP RC522 (13.56MHz)
  • Portachiavi / tessere / braccialetti standard ISO / IEC 14443-A
  • Cicalino
  • 2x LED

    • Principio di funzionalità hardware:

  • Il lettore RF5 RC522 comunica con Arduino tramite il bus SPI. Legge l'UID (identificatore) delle schede RFID nell'intervallo, inoltra questo identificatore ad Arduino, che esegue la logica del programma. Il lettore risponde alla distanza del chip di circa 3 cm dall'antenna.
  • Arduino confronta l'indirizzo della scheda di lettura in una condizione multipla, applica gli stati digitali ai diodi, esegue un contatore universale ogni 100 ms (conta in decimi di secondo), controlla il conteggio di entrambe le volte.
  • L'ora viene inviata tramite il bus I2C al display dei caratteri 20x4, dove viene tracciata. Il display dura fino a 30 ms a una velocità del bus I2C di 100 kHz - Velocità standard.
    • Intervallo di tempo massimo che può essere elencato nel sistema: 99 ore, 59 minuti e 59 secondi
    Consumo: circa 110 mAh con alimentazione 5V. Comodo da alimentare tramite power bank, caricatore da un telefono con uscita USB.

#include <SPI.h>
#include <MFRC522.h>
#define RST_PIN         9
#define SS_1_PIN        10
#define NR_OF_READERS   1
byte ssPins[] = {SS_1_PIN};
MFRC522 mfrc522[NR_OF_READERS];
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    mfrc522[reader].PCD_Init(ssPins[reader], RST_PIN);
    mfrc522[reader].PCD_SetAntennaGain(112); // set to max (00001110)
    Serial.print(F("Reader "));
    Serial.print(reader);
    Serial.print(F(": "));
    mfrc522[reader].PCD_DumpVersionToSerial();
  }
}

void loop() {
  for (uint8_t reader = 0; reader < NR_OF_READERS; reader++) {
    if (mfrc522[reader].PICC_IsNewCardPresent() && mfrc522[reader].PICC_ReadCardSerial()) {
      Serial.print(F("Reader "));
      Serial.print(reader);
      Serial.print(F(": Card UID:"));
      dump_byte_array(mfrc522[reader].uid.uidByte, mfrc522[reader].uid.size);
      Serial.println();
      Serial.print(F("PICC type: "));
      MFRC522::PICC_Type piccType = mfrc522[reader].PICC_GetType(mfrc522[reader].uid.sak);
      Serial.println(mfrc522[reader].PICC_GetTypeName(piccType));
      mfrc522[reader].PICC_HaltA();
      mfrc522[reader].PCD_StopCrypto1();
    }
  }
}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Nacitany kod: ");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.println(kod);
}

    Scanner I2C - programma per leggere l'indirizzo del convertitore I2C per display LCD

    Uscita scanner I2C prevista:

    La libreria Wire è integrata nell'IDE di Arduino, non è richiesta alcuna installazione. Il programma può essere caricato immediatamente. Si consiglia di collegare il convertitore hardware - I2C per display 16x2 / 20x4, anche senza display, prima di avviare il programma. 
    #include <Wire.h>
 
 
void setup()
{
  Wire.begin();
 
  Serial.begin(9600);
  while (!Serial);             // Leonardo: wait for serial monitor
  Serial.println("\nI2C Scanner");
}
 
 
void loop()
{
  byte error, address;
  int nDevices;
 
  Serial.println("Scanning...");
 
  nDevices = 0;
  for(address = 1; address < 127; address++ )
  {
    // The i2c_scanner uses the return value of
    // the Write.endTransmisstion to see if
    // a device did acknowledge to the address.
    Wire.beginTransmission(address);
    error = Wire.endTransmission();
 
    if (error == 0)
    {
      Serial.print("I2C device found at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.print(address,HEX);
      Serial.println("  !");
 
      nDevices++;
    }
    else if (error==4)
    {
      Serial.print("Unknown error at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.println(address,HEX);
    }    
  }
  if (nDevices == 0)
    Serial.println("No I2C devices found\n");
  else
    Serial.println("done\n");
 
  delay(5000);           // wait 5 seconds for next scan
}

    Programma di test (Shareware) - Button DOMINATOR con codice macchina

    È disponibile una versione Shareware (.hex) per il progetto Button DOMINATOR. La versione consente di misurare un massimo di 15 secondi di gioco per ogni squadra. L'obiettivo della versione è verificare la compatibilità con il tuo hardware. I codici macchina basati sull'indirizzo I2C del convertitore LCD sono disponibili nel repository Github. Gli indirizzi più utilizzati sono 0x27 e 0x3F. Se non conosci l'indirizzo del tuo convertitore, usa l'esempio integrato in Arduino IDE -> I2C Scanner, che scriverà l'indirizzo sull'interfaccia UART (monitor seriale). Il codice macchina può essere caricato sul microcontrollore Arduino Uno tramite il software AVRDUDE - parte dell'IDE Arduino, o con un'interfaccia GUI tramite XLoader - più facile da usare.

  • AVRDUDE deve essere eseguito tramite il CMD (riga di comando) installato con l'IDE di Arduino. Situato in C: /Users/User/Documents/ArduinoData/packages/arduino/tools/avrdude/6.X.X-arduino/bin
    • Comando di esempio per caricare un programma su Arduino Uno / Nano (AtMega328P):
  • avrdude -CC: avrdude.conf -v -patmega328p -carduino -PCOM21 -b115200 -D -Uflash: w: sketch.ino.hex: i
  • XLoader è un'applicazione Windows (strumento eseguibile .exe) con una semplice interfaccia grafica , che consente la selezione del firmware - codice macchina (.hex), la piattaforma di destinazione e può mostrare dinamicamente il COM porta su cui si trova il dispositivo.
  • Facendo semplicemente clic su Carica, il codice macchina può essere caricato automaticamente (XLoader utilizza lo strumento AVRDUDE simile all'IDE di Arduino). La finestra XLoader informa dell'avvenuto caricamento sotto forma di un elenco di byte caricati.
    • Il firmware può essere scaricato dal repository del progetto: QUI . Il firmware si trova nella cartella / firmware / diviso in base all'indirizzo del convertitore I2C (che hai scoperto dall'output dello scanner I2C sull'interfaccia UART).
