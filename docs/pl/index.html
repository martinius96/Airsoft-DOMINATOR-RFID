    Opis projektu RFID DOMINATOR dla Airsoft:

    DOMINATOR (Domination Timer) to urządzenie - stoper do sportów Airsoft, Paintball z mikrokontrolerem Arduino (Uno/Nano) z układem AtMega328P AVR. Stoper posiada wyświetlacz znakowy LCD o wymiarach 16x2 (16 znaków x 2 linie) lub 20x4, na którym wyświetlane są czasy poszczególnych drużyn. W grze są 2 drużyny - RED team i GRE team które grają przeciwko sobie i zajmują wyimaginowany punkt, który jest tworzony przez to urządzenie - DOMINATOR. Na początku gry wyświetlacz pokazuje czasy 00 godzin, 00 minut, 00 sekund dla RED team , Zespół GRE < . Każdy członek zespołu wyposażony jest w kartę, bransoletkę lub breloczek, który komunikuje się na częstotliwości 13,56 MHz z czytnikiem RFID, jeśli znajduje się w jego zasięgu. Breloki i bransoletki mogą być oznaczone kolorami (zielony, czerwony w zależności od zespołu i funkcji…), co pozwala na bardziej efektywne rozdawanie ich graczom, ale także ich archiwizację i przechowywanie. Jeśli drużyna RED zajmuje punkt, gracz drużyny dołącza swoją kartę do czytnika, tym samym zajmując punkt. Ta akcja uruchamia brzęczyk, który informuje o zmianie punktu. Następnie dioda LED drużyny RED na tej stacji zaświeci się, a wyświetlacz zacznie dodawać czas dla tej drużyny. Jeśli drużyna GRE zajmie punkt, a członek tej drużyny przyłoży kartę do czytnika, brzęczyk zabrzmi ponownie, dioda LED tej drużyny zapali się, czas drużyny RED zostanie wstrzymany i Liczy się czas drużyny. GRE, która zajęła punkt. Jeśli karta / brelok jest dołączony przez organizatora (sędziego), oba czasy są wstrzymane do momentu, gdy jedna z drużyn włoży kartę. Ten typ karty nadaje się zatem do rozpoczęcia przerwy, ale może być również użyty na koniec gry, kiedy konieczne jest zatrzymanie czasu, aby ocenić grę. Jeśli włożona jest karta z Erasera, oba czasy są zatrzymywane i resetowane do początkowej wartości 00 godzin, 00 minut, 00 sekund, która jest wyświetlana. Karta Eraser służy do ponownego uruchomienia gry, zainicjowania nowej gry bez konieczności włączania cyklu. Projekt nadaje się na imprezy Airsoft, obiekty sportowe, czy zespoły Airsoft, które mogą zbudować dane urządzenie Dominator z funkcjonalnym oprogramowaniem, do którego jest schemat elektryczny i projekt PCB do produkcji maszyn z wierceniem. Zaletą projektu RFID DOMINATOR jest to, że licencja umożliwia klonowanie i tworzenie X punktów z tą samą konfiguracją za pomocą kodu maszynowego. Możliwe jest więc umieszczenie w grze dowolnej liczby niezależnie działających punktów RFID DOMINATOR, które reagują na identyczne karty RFID. System jest przetestowany, funkcjonalny, można go szybko wdrożyć na kompatybilnym sprzęcie. Czytnik NXP RC522 pracuje na częstotliwości 13,56 MHz, rejestruje tagi RFID z odległości ok. 3 centymetrów (wymagany jest prawie fizyczny kontakt z czytnikiem), zgodny format tagów ISO/IEC 14443-A (MIFARE CLASSIC 1K, DesFire EV1, kompatybilność wsteczna z ISO/IEC 14443 -4). Podany standard tagów obejmuje również ISIC, karty magistrali, które pełnią funkcję kart NFC i mogą przekazywać swój adres fizyczny - UID do czytnika znajdującego się w zasięgu.

    Zasada funkcjonalności projektu RFID DOMINATOR od strony sprzętowej:

    Czytnik RFID NXP RC522 komunikuje się z mikrokontrolerem (układ Arduino/autonomiczny AtMega328P) za pośrednictwem magistrali SPI z sygnałem zegarowym (CLK) 4 MHz. Czytnik reaguje na odległość chipa około 3 cm od anteny, natomiast posiada konfigurowalny zysk anteny w zakresie od 18 do 48 dB (w aplikacji finalnej używane jest 48 dB). Odczytuje UID (identyfikator) kart RFID znajdujących się w zasięgu, przekazuje ten identyfikator do Arduino. Arduino porównuje adres odczytanej karty w wielu warunkach, stosuje stany cyfrowe do diod, uruchamia uniwersalny licznik sterowany funkcją millis(), który odpowiada za zliczanie obu czasów. W oparciu o spełniony warunek licznik zostanie zastosowany do danej drużyny, do której należy UID. Czas jest przesyłany magistralą I2C na wyświetlacz 20x4 znakowy, gdzie jest wykreślany w określonej linii - tylko określona pozycja jest nadpisywana w celu jak najszybszego odzyskania czasu. Wypisanie wszystkich ustawień wyświetlacza zajmuje do 30ms przy prędkości magistrali I2C wynoszącej 100 kHz - standardowa prędkość, która jest używana. W przypadku dużego zainteresowania możliwe jest stworzenie oprogramowania w kodzie maszynowym dla AtMega328P do pomiaru dla innych typów czytników RFID: RDM6300, PN532, Wiegand W26, zwiększenie sygnału zegara magistrali I2C do 400 kHz (Fast Speed).

    Maksymalny zakres czasu, który może być wymieniony w systemie: 99 godzin, 59 minut i 59 sekund
    Zużycie: ok. 110mAh przy zasilaniu 5V. Wygodny do zasilania przez power bank, ładowarkę z telefonu z wyjściem USB.

    DOMINATOR RFID nadaje się do:

    Projekt RFID DOMINATOR pod względem swojej wszechstronności nadaje się do wdrożenia w akcjach Airsoft, a dzięki możliwości klonowania oprogramowania istnieje możliwość zainstalowania X punktów DOMINATOR na polu gry z identycznym oprogramowaniem, w które gracze mogą grać z te same karty RFID. DOMINATOR jest również odpowiedni dla drużyn, boisk sportowych i placów zabaw, które mogą wykorzystać to urządzenie do rozszerzenia usług i trybów gry oferowanych dla różnej liczby graczy. Jeśli karty, bransoletki lub breloki RFID są oznaczone kolorami, jest to bardzo łatwe i uniwersalne rozprowadzanie wśród graczy w oparciu o ich drużynę. Ich archiwizacja jest również prosta. Korzystanie z kart, bransoletek lub breloczków jest łatwe i poradzi sobie w grach Airsoft w każdym wieku. Sam chip RFID jest chroniony w plastikowej obudowie, nie przeszkadza mu brud, woda, jest w pełni bezobsługowy. Nie posiada baterii, chip aktywuje czytnik RFID, który ładuje go transmitowanym polem elektromagnetycznym i umożliwia wysłanie UID jeśli znajduje się w zasięgu czytnika RFID. DOMINATOR nadaje się do trybów gry z taką samą liczbą graczy w obu drużynach, m.in. DOMINACJA, Punkt kontrolny (Punkty za wiele sklonowanych punktów DOMINATORA). Stosowany jest głównie w sportach Airsoft, grach w Paintball.

    Program do odczytu adresów UID kart RFID dla DOMINATORA

    Program slúži na načítanie identifikátorov (UID) RFID tagov, kariet, ktoré budú použité v programe pre RFID DOMINATOR - tento zdrojový kód nereprezentuje RFID DOMINATOR, ani jeho logiku!. Výpis UID RFID karty sa realizuje do sériového (UART) monitoru - použitá rýchlosť 9600 baud/s (znakov za sekundu). Výpis je vhodné ukladať a prepísať do prehľadného .xls (EXCEL) dokumentu, ktorý uľahčí správu priradených RFID kariet ku konkrétnym tímom a entitám pre finálnu aplikáciu RFID DOMINATOR, ktorej strojový kód je viazaný na dodané UID kariet. Pre úspešné nahratie zdrojového kódu pre načítanie UID kariet je nutné importovať knižnicu MFRC522 do vášho prostredia Arduino IDE. Knižnica MFRC522 vo verzii 1.4.8: Github repozitár - stiahnuť .zip archív a obsah archívu nakopírovať do Arduino/libraries (zložka sa nachádza najčastejšie v Dokumentoch, alebo podľa vlastnej inštalácie v inom umiestnení).

/*
 * Typical pin layout used:
 * -----------------------------------------------------------------------------------------
 *             MFRC522      Arduino       Arduino   Arduino    Arduino          Arduino
 *             Reader/PCD   Uno/101       Mega      Nano v3    Leonardo/Micro   Pro Micro
 * Signal      Pin          Pin           Pin       Pin        Pin              Pin
 * -----------------------------------------------------------------------------------------
 * RST/Reset   RST          9             5         D9         RESET/ICSP-5     RST
 * SPI SS      SDA(SS)      10            53        D10        10               10
 * SPI MOSI    MOSI         11 / ICSP-4   51        D11        ICSP-4           16
 * SPI MISO    MISO         12 / ICSP-1   50        D12        ICSP-1           14
 * SPI SCK     SCK          13 / ICSP-3   52        D13        ICSP-3           15
 *
 */
#include <SPI.h>
#include <MFRC522.h>
#define SS_PIN 10
#define RST_PIN 9

MFRC522 rfid(SS_PIN, RST_PIN); // Instance of the class
void setup() {
  Serial.begin(9600);
  while (!Serial);
  SPI.begin();
  rfid.PCD_Init(); // Init MFRC522
  rfid.PCD_SetAntennaGain(112); // set to max (00001110)
  rfid.PCD_DumpVersionToSerial();
}

void loop() {
  if (rfid.PICC_IsNewCardPresent() && rfid.PICC_ReadCardSerial()) {
    Serial.print(F("Card UID: "));
    dump_byte_array(rfid.uid.uidByte, rfid.uid.size);
    Serial.println();
    Serial.print(F("PICC type: "));
    MFRC522::PICC_Type piccType = rfid.PICC_GetType(rfid.uid.sak);
    Serial.println(rfid.PICC_GetTypeName(piccType));
    rfid.PICC_HaltA();
    rfid.PCD_StopCrypto1();
  }

}

void dump_byte_array(byte *buffer, byte bufferSize) {
  for (byte i = 0; i < bufferSize; i++) {
  }
  Serial.print("Nacitany kod: ");
  unsigned long kod = 10000 * buffer[4] + 1000 * buffer[3] + 100 * buffer[2] + 10 * buffer[1] + buffer[0]; //finalny kod karty
  Serial.println(kod);
}

    Optimalizácia programu pre RFID DOMINATOR:

    Čísla UID RFID kariet pre RED, GRE, rozhodcu a erasera vložené do polí, ktoré môžu byť ľahko škálovateľné pre rozšírenie počtu hráčov, ktorí sú priradený ku konkrétnej skupine. Celé polia sú uložené do flash pamäte mikrokontroléru ako rad-only segmenty, teda polia nemusia bežať v RAM pamäti zaberať tak operačnú pamäť, ktorú DOMINATOR využíva. Štandardný program RFID DOMINATOR-a pre 10x UID pre RED i GRE tím a 2x UID pre Rozhodcu a Erasera zaberá 9512 bajtov flash pamäte, t.j. 30% a 501 bajtov pamäte RAM, t.j. 24%. Statické reťazce (const) premenné uložené do PROGMEM. Použité F makro pre statické výpisy na UART monitor, prípadne na LCD znakový displej.

    I2C Scanner - program pre načítanie adresy I2C prevodníka pre LCD displej

    Očakávaný výstup programu I2C Scanner:

    Knižnica Wire je vstavaná do Arduino IDE, nie je nutná jej inštalácia. Program je možné skompilovať a overiť komunikáciu po I2C zbernici s LCD znakovým displejom. I2C prevodník je vhodné pripojiť k Arduinu na hardvérové I2C vývody (SCL na A5, SDA na A4) ešte pred nahratím programu, stačí aj bez dispelja. Načítanú adresu displeja je nutné dodať pre úspešný build pre Arduino spolu s UID RFID kariet. 

    #include <Wire.h>
 
 
void setup()
{
  Wire.begin();
 
  Serial.begin(9600);
  while (!Serial);             // Leonardo: wait for serial monitor
  Serial.println("\nI2C Scanner");
}
 
 
void loop()
{
  byte error, address;
  int nDevices;
 
  Serial.println("Scanning...");
 
  nDevices = 0;
  for(address = 1; address < 127; address++ )
  {
    // The i2c_scanner uses the return value of
    // the Write.endTransmisstion to see if
    // a device did acknowledge to the address.
    Wire.beginTransmission(address);
    error = Wire.endTransmission();
 
    if (error == 0)
    {
      Serial.print("I2C device found at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.print(address,HEX);
      Serial.println("  !");
 
      nDevices++;
    }
    else if (error==4)
    {
      Serial.print("Unknown error at address 0x");
      if (address < 16)
        Serial.print("0");
      Serial.println(address,HEX);
    }    
  }
  if (nDevices == 0)
    Serial.println("No I2C devices found\n");
  else
    Serial.println("done\n");
 
  delay(5000);           // wait 5 seconds for next scan
}
